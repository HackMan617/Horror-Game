# NIGHTMARE REVEAL — 64px hi-bit scare sheets & customization drain

The close-up tier of the **WRUNG** nightmare player — the thing that flashes where you should
be, now big enough to hurt. A significant style step up from the 32px rig sheets (which are
**unchanged**, see `NIGHTMARE_PLAYER.md`): semi-real proportions, rich shading ramps, wet
specular viscera, sickly rim light, deep blacks — still point-filter pixel art. The body is
**rotting from the inside**: withered gaunt skin, coherent decay lesions, threadveins, a
slipping mask over wrong flesh, barbed wire, a burst belly and guts it drags behind itself.

**And it wears the player.** The sheets are baked in exact *key colors* for skin / hair /
shirt / pants / eye pin-light. Remap those keys to a **drained** version of the player's
Character-Creator palette and the horror wears *their* character. `Nightmare Reveal.dc.html`
does this live (swatches + auto-loads the saved `charCreatorSel`).

### Rows — the four clips (10 frames each)

```
row 0  FLASH      anticipation crouch → lunges AT the camera; the face
                  and claw fill the cell, maw open, pin-lights on      — play once, ~70 ms/f, then CUT away
row 1  SCREAM     the jaw unhinges too far, mask peels wider, a hand
                  claws up its own skull; ends held on the full maw    — play once, ~95 ms/f, hold 9
row 2  SNAP       head hangs chin-to-chest (0–3), SNAPS up with an
                  overshoot (4–5), then the dead stare, eyes lit (6–9) — play once, ~95 ms/f, hold 9
row 3  CRAWL      prone, dragging itself toward you on its forearms,
                  guts trailing, blood smear ticking backward          — seamless LOOP, ~110 ms/f
```

Sprite index = `row * 10 + col`. The stare tremble (SNAP 6–9) and glitch/red-wash punch are
**engine transforms** — keep owning those at playback (see `NIGHTMARE_PLAYER.md`); the frames
carry the anatomy.

### Both tiers share layout

`base` (doppelgänger) = drained + withered + weeping, faint seam, one wire bite, **no** burst
belly / spine / claw. `you` (the reveal) = everything: gut rope, bone knobs, hooks, embers,
full mask peel. Same rows/cols, so one animator handles either texture.

## The customization drain — keys & recipe

Every customizable material is baked as EXACT hexes (never blended). Remap these and nothing
else; all gore/rot/bone/wire colors are fixed and off-limits.

| Material | Keys (hi / base / sh / deep) |
|---|---|
| skin  | `#ffe0c4` `#f0b890` `#c07a4c` `#8f5432` |
| hair  | `#c98a4a` `#9c5a26` `#5e3410` |
| shirt | `#f06a5a` `#d83030` `#982018` |
| pants | `#6a8ae8` `#3a5bd0` `#26398c` |
| eye pin-light | `#00e2ff` (SNAP 6–9, FLASH mid+ frames) |

The base/sh keys for skin·hair·shirt·pants are the **same source colors the Character
Creator swaps**, so the same selection data drives both. Build the target palette from the
player's chosen colors, *drained*:

```js
// chosen = {skin:{base,sh}, hair:{base,sh}, shirt:{base,sh}, pants:{base,sh}, eye:{base}}
const mix=(a,b,t)=>a.map((v,i)=>v+(b[i]-v)*t);            // rgb arrays
const scale=(a,f)=>a.map(v=>v*f);
const dSkin=c=>mix(c, rgb('#c9c2b2'), 0.55);              // toward wax
const dHair=c=>mix(c, rgb('#8a8478'), 0.45);              // toward ash
const dFab =c=>mix(c, rgb('#6a655c'), 0.42);              // toward grave-cloth
map['#ffe0c4']=scale(dSkin(chosen.skin.base),1.16); map['#f0b890']=dSkin(chosen.skin.base);
map['#c07a4c']=dSkin(chosen.skin.sh);               map['#8f5432']=scale(dSkin(chosen.skin.sh),0.72);
map['#c98a4a']=scale(dHair(chosen.hair.base),1.15); map['#9c5a26']=dHair(chosen.hair.base);
map['#5e3410']=dHair(chosen.hair.sh);
// shirt & pants: same shape with dFab, hi = scale(base,1.18)
map['#00e2ff']=mix(rgb(chosen.eye.base), rgb('#dff0ff'), 0.5);   // pin-light keeps their eye color
```

Apply as an exact-match LUT over the sheet once per selection (the viewer caches one cooked
canvas). **If you skip the remap, remap the eye key anyway** — raw `#00e2ff` is a deliberate
placeholder cyan.

`Nightmare Reveal.dc.html` reads the Creator's saved selection from
`localStorage.charCreatorSel` on load ("wearing YOUR saved character") and offers the full
swatch set for directing scares by hand.

## Using the clips

- **The queue scare** (see `NIGHTMARE_PLAYER.md` for the 32px flow): when the beat lands,
  swap the queued NPC for this 64px sprite scaled to the same footprint and play **FLASH**;
  cut back to normal on the last frame. No ease-out — it's just gone.
- **SNAP** is the *noticing* beat for a doppel that's been standing in the world on `base`:
  play SNAP on the `base` sheet, then swap to `you` mid-stare for the gore turn.
- **SCREAM** is the hold-scare (cutscenes, mirrors, doorways). Pair with a wet jaw-crack SFX
  at frame ~4.
- **CRAWL** loops — corridor pursuit at floor level. It should be slightly too slow to
  escape from badly, and never faster than the player.
- Match `body` to the player's archetype and feed the drain map from their save. The horror
  is that it's *them*.

## Re-exporting

```js
eval(await readFile('nightmarerevealgen.js'));
await window.NightmareReveal.buildAll({ createCanvas, saveFile });            // all 4 sheets
await window.NightmareReveal.buildAll({ createCanvas, saveFile, only:'male_you' });
```

Tunables in `nightmarerevealgen.js`: `KEY` (remap keys) / `G` (fixed gore palette); `head()`
(orbits, maw, seam/peel, hair incl. gravity falls); `standing()` / `crawl()` (anatomy & part
order); `rotPass` (decay lesions + cheek lesion + threadveins); `gutRope` / `wireBand` /
`claw` / `emberCracks`; and `frameParams` (the per-clip schedule: lunge scale/anchor, jaw
open, snap overshoot, crawl cycle).
